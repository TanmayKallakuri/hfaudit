# Security Policy

## Reporting Vulnerabilities in HFAudit

If you discover a security vulnerability in the HFAudit tool itself (the scanner, dashboard, or infrastructure), please report it responsibly.

### Contact

Email: **security@hfaudit.dev**

Please include:
- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix or mitigation:** Best effort, typically within 30 days

### Scope

This security policy covers vulnerabilities in:
- The HFAudit scanner (`scanner/`)
- The HFAudit dashboard (`web/`)
- The backend infrastructure (`infra/`)
- CI/CD pipelines and build processes

### Out of Scope

**Findings about malicious HuggingFace models** are not vulnerabilities in HFAudit. If you have discovered a malicious model on HuggingFace, please report it directly to HuggingFace at security@huggingface.co or use HFAudit's disclosure pipeline as described in [RESPONSIBLE_DISCLOSURE.md](RESPONSIBLE_DISCLOSURE.md).

### Recognition

We credit all reporters in our security advisories (unless you prefer to remain anonymous).
